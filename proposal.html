  • TMDB api for TV Shows and Movies
  • AniList for Anime and Manga
  • Openlibrary api for Books

How will this data be used:

The data fetched from API is be used to provide showtimes, watch providers, reviews and number of episodes etc to the end user.

Simplified Database Schema

  • users: userID, username, password, email
  • videoProfile: videoID, seriesID, userID
  • textProfile: textID, seriesID, userID
  • profileData: userID, username, password, email
  • review: userID, reviewID, mediaID, mediaType, reviewDate, comments, stars
  • textSeries: seriesID, title, author, publisher, publishDate, link
  • text: textID, seriesID, isbn, textID, title, iteration, link
  • videoSeries: seriesID, title, author, publisher, publishDate, link
  • video: videotitle, seriesID, director, publisher, publishDate, link

User Demographic and Data:

Problems Statement

There are too many streaming apps and media apps. We wanted an easy way to track all of the media we consume.

Target Demographic

This app is geared towards the average consumer of streamed media {movies, Shows, Books, Comics}

Goals

This app will allow users enter media forms that they are interested.
They will also be able to track media as it updates.
